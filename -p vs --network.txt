Here is a clear, simple, interview-friendly explanation of the difference between
`-p` (port mapping) and `--network host` (host network mode):
------------------------------------------------------------------------------------------------
------------------------------------------------------------------------------------------------
------------------------------------------------------------------------------------------------
‚úÖ 1. `-p` (Port Mapping)
Purpose: Map container ports to host ports.

How it works:
Your container runs in an isolated network.
To make it reachable from your host machine, you map ports:
docker run -p 3000:3000 myapp

This means:
* Container port: 3000
* Host port: 3000
Traffic goes through Docker's virtual network layer.

Key Points:
‚úî Secure (container and host remain isolated)
‚úî You can choose any host port
‚úî Works on Linux, Mac, Windows

Interview line:
> `-p` publishes a container port to the host by using Docker's NAT and bridge network.
------------------------------------------------------------------------------------------------
------------------------------------------------------------------------------------------------
------------------------------------------------------------------------------------------------
‚úÖ 2. `--network host` (Host Network Mode)
Purpose: Container uses the host's network directly (no isolation).

How it works:
docker run --network host myapp

This means:
* No port mapping required
* Host and container share the same network stack
* Container uses host IP directly
* If your app listens on port 3000, it is automatically available on host:3000

Key Points:
‚úî No need for `-p`
‚úî Better performance (no NAT)
‚ùå Less secure (no network isolation)
‚ùå Only works fully on Linux
(Windows/Mac use a VM so behavior differs)

Interview line:
> `--network host` removes Docker's network isolation and makes the container use the host's network directly, which eliminates the need for port mapping.
------------------------------------------------------------------------------------------------

------------------------------------------------------------------------------------------------
üîé When to use which?

Use `-p` when:
* Normal web apps
* Production environments
* You want isolation
* You want flexibility for port mapping

Use `--network host` when:
* High performance networking needed
* Running monitoring tools (Prometheus, ELK)
* Running local development tools
* You trust the container
